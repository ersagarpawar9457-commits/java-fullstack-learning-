# ‚òï JAVA DEFINITIONS

---

## Core Concepts

| Term | Definition |
|------|------------|
| **JVM** | Java Virtual Machine ‚Äî executes Java bytecode, makes Java platform-independent |
| **JDK** | Java Development Kit ‚Äî tools to develop Java programs (compiler, JRE, debugger) |
| **JRE** | Java Runtime Environment ‚Äî provides libraries and JVM to run Java programs |
| **Bytecode** | Intermediate code generated by Java compiler (.class files), executed by JVM |
| **Compiler** | Converts source code (.java) to bytecode (.class) |

---

## Variables & Data Types

| Term | Definition |
|------|------------|
| **Variable** | A named storage location in memory that holds a value |
| **Primitive Types** | Basic data types: `int`, `double`, `boolean`, `char`, `byte`, `short`, `long`, `float` |
| **Reference Types** | Objects and arrays ‚Äî store memory address, not actual value |
| **Type Casting** | Converting one data type to another (implicit or explicit) |
| **Wrapper Class** | Object representation of primitives (Integer, Double, Boolean, etc.) |
| **Autoboxing** | Automatic conversion of primitive to wrapper object |
| **Unboxing** | Automatic conversion of wrapper object to primitive |

---

## Control Flow

| Term | Definition |
|------|------------|
| **Conditional Statement** | Code that executes based on a condition (if-else, switch) |
| **Loop** | Repeated execution of code block (for, while, do-while) |
| **Break** | Exits the current loop or switch immediately |
| **Continue** | Skips current iteration and moves to next loop iteration |
| **Iteration** | One complete cycle/execution of a loop |

---

## OOP (Object-Oriented Programming)

| Term | Definition |
|------|------------|
| **Class** | Blueprint/template for creating objects; defines properties and behaviors |
| **Object** | Instance of a class; has state (fields) and behavior (methods) |
| **Constructor** | Special method that initializes an object when created (same name as class) |
| **Method** | A function defined inside a class that performs an action |
| **Field/Attribute** | Variable declared inside a class that holds object state |
| **this** | Reference to the current object |
| **static** | Belongs to the class, not to any specific object instance |
| **final** | Cannot be changed ‚Äî final variable (constant), final method (no override), final class (no extend) |

### 4 Pillars of OOP

| Term | Definition |
|------|------------|
| **Encapsulation** | Bundling data and methods together; hiding internal details using access modifiers |
| **Inheritance** | Child class acquires properties and methods of parent class (`extends`) |
| **Polymorphism** | Ability to take many forms ‚Äî same method name, different implementations |
| **Abstraction** | Hiding complex implementation, showing only essential features |

### Related Terms

| Term | Definition |
|------|------------|
| **Access Modifiers** | `public` (everywhere), `private` (same class), `protected` (same package + subclass), `default` (same package) |
| **Method Overloading** | Same method name, different parameters (compile-time polymorphism) |
| **Method Overriding** | Child class redefines parent's method (runtime polymorphism) |
| **Abstract Class** | Class with `abstract` keyword; cannot be instantiated; may have abstract methods |
| **Abstract Method** | Method without body; must be implemented by child class |
| **Interface** | Contract defining methods a class must implement; 100% abstraction |
| **super** | Reference to parent class; used to call parent constructor or method |

---

## Arrays

| Term | Definition |
|------|------------|
| **Array** | Fixed-size, ordered collection of elements of the same type |
| **Index** | Position of element in array (starts from 0) |
| **Length** | Number of elements an array can hold (`array.length`) |
| **ArrayIndexOutOfBoundsException** | Error when accessing invalid index |

---

## Collections Framework

| Term | Definition |
|------|------------|
| **Collection** | Interface representing a group of objects |
| **List** | Ordered collection that allows duplicates (ArrayList, LinkedList) |
| **Set** | Collection that does NOT allow duplicates (HashSet, TreeSet) |
| **Map** | Key-value pairs; keys are unique (HashMap, TreeMap) |
| **Queue** | FIFO (First In First Out) collection |
| **Iterator** | Object to traverse through collection elements |
| **Generics** | Type parameters that provide compile-time type safety (`List<String>`) |

### Implementations

| Term | Definition |
|------|------------|
| **ArrayList** | Resizable array; fast random access O(1), slow insert/delete O(n) |
| **LinkedList** | Doubly-linked list; slow access O(n), fast insert/delete at ends O(1) |
| **HashSet** | Uses hashing; no order; O(1) operations |
| **TreeSet** | Sorted set; O(log n) operations |
| **HashMap** | Key-value pairs using hashing; O(1) operations |
| **TreeMap** | Sorted by keys; O(log n) operations |

---

## Exception Handling

| Term | Definition |
|------|------------|
| **Exception** | An event that disrupts normal program flow |
| **Error** | Serious problem that application shouldn't try to handle (OutOfMemoryError) |
| **Checked Exception** | Must be handled (try-catch) or declared (throws); detected at compile-time |
| **Unchecked Exception** | RuntimeException; optional to handle; detected at runtime |
| **try** | Block containing code that might throw exception |
| **catch** | Block that handles the exception |
| **finally** | Block that always executes (cleanup code) |
| **throw** | Keyword to explicitly throw an exception |
| **throws** | Declares that method might throw an exception |
| **Stack Trace** | List of method calls that shows where exception occurred |

---

## Multithreading

| Term | Definition |
|------|------------|
| **Thread** | Lightweight sub-process; smallest unit of execution |
| **Multithreading** | Concurrent execution of multiple threads |
| **Process** | Independent program with its own memory space |
| **Runnable** | Interface with `run()` method; preferred way to create threads |
| **start()** | Method that creates new thread and calls run() |
| **run()** | Contains the code to be executed by thread |
| **Synchronization** | Controlling thread access to shared resources |
| **Race Condition** | Bug when multiple threads access shared data without synchronization |
| **Deadlock** | Two or more threads waiting for each other forever |
| **Thread Pool** | Group of reusable threads (ExecutorService) |
| **wait()** | Causes thread to wait until notified |
| **notify()** | Wakes up one waiting thread |

# üé® HTML & CSS DEFINITIONS

---

## HTML

| Term | Definition |
|------|------------|
| **HTML** | HyperText Markup Language ‚Äî structure of web pages |
| **Tag** | Markup element enclosed in `<>` (e.g., `<p>`, `<div>`) |
| **Element** | Complete tag with content: `<p>content</p>` |
| **Attribute** | Additional info about element: `<a href="url">` |
| **Semantic HTML** | Tags that describe meaning (`<header>`, `<nav>`, `<article>`) |
| **Block Element** | Takes full width, starts new line (`<div>`, `<p>`, `<h1>`) |
| **Inline Element** | Takes only needed width, no new line (`<span>`, `<a>`, `<strong>`) |
| **DOM** | Document Object Model ‚Äî tree representation of HTML structure |

---

## CSS

| Term | Definition |
|------|------------|
| **CSS** | Cascading Style Sheets ‚Äî styling of web pages |
| **Selector** | Targets HTML elements to style (`p`, `.class`, `#id`) |
| **Property** | What to style (`color`, `font-size`, `margin`) |
| **Value** | How to style (`red`, `16px`, `10px`) |
| **Specificity** | Priority system: inline > ID > class > element |
| **Cascade** | Order matters ‚Äî later rules override earlier ones |
| **Inheritance** | Child elements inherit some parent styles |

### Box Model

| Term | Definition |
|------|------------|
| **Content** | Actual content of the element (text, image) |
| **Padding** | Space between content and border (inside) |
| **Border** | Line around the padding |
| **Margin** | Space outside the border (between elements) |
| **box-sizing: border-box** | Width/height includes padding and border |

### Flexbox

| Term | Definition |
|------|------------|
| **Flexbox** | 1D layout system (row OR column) |
| **Flex Container** | Parent element with `display: flex` |
| **Flex Item** | Direct children of flex container |
| **Main Axis** | Primary direction (row = horizontal, column = vertical) |
| **Cross Axis** | Perpendicular to main axis |
| **justify-content** | Alignment along MAIN axis |
| **align-items** | Alignment along CROSS axis |
| **flex-grow** | How much item should grow relative to others |
| **flex-shrink** | How much item should shrink relative to others |

### CSS Grid

| Term | Definition |
|------|------------|
| **CSS Grid** | 2D layout system (rows AND columns) |
| **Grid Container** | Parent element with `display: grid` |
| **Grid Item** | Direct children of grid container |
| **grid-template-columns** | Defines column sizes |
| **grid-template-rows** | Defines row sizes |
| **fr** | Fraction unit ‚Äî flexible sizing |
| **gap** | Space between grid items |

### Pseudo-classes & Pseudo-elements

| Term | Definition |
|------|------------|
| **Pseudo-class** | Selects element in specific STATE (`:hover`, `:focus`, `:first-child`) |
| **Pseudo-element** | Selects PART of element (`::before`, `::after`, `::first-letter`) |
| **:hover** | When mouse is over element |
| **:focus** | When element is focused (input, button) |
| **:nth-child(n)** | Selects nth child element |
| **::before** | Inserts content before element |
| **::after** | Inserts content after element |
